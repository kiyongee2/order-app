# PRD - 관리자 화면

## 1. 개요

### 1.1 화면명
관리자 (Admin Dashboard)

### 1.2 목적
관리자가 실시간으로 주문을 확인하고, 재고를 관리하며, 주문 상태를 업데이트할 수 있는 관리 화면

### 1.3 우선순위
매우 높음 (주요 기능)

### 1.4 접근 권한
현재는 제한 없음 (추후 인증 추가 가능)

---

## 2. 기능 요구사항

### 2.1 상단 네비게이션 (Header)
- **앱 로고/이름**: "COZY" 표시
- **탭 메뉴**: 
  - "주문하기" (주문 화면으로 이동)
  - "관리자" (현재 화면)
- **레이아웃**: 상단 고정 네비게이션

### 2.2 관리자 대시보드 (Dashboard Summary)
위치: 화면 상단, 재고 현황 위에 위치

#### 2.2.1 주요 지표 표시
한 줄로 주문 상태의 빠른 요약 제시

- **총 주문 수**: 전체 주문 수
  - 예: "총 주문 1"
- **주문 접수**: 신규 접수된 주문 수
  - 예: "주문 접수 1"
- **제조 중**: 현재 제조 중인 주문 수
  - 예: "제조 중 0"
- **제조 완료**: 완성된 주문 수
  - 예: "제조 완료 0"

#### 2.2.2 시각 디자인
- 박스 형식으로 표시
- 슬래시(/)로 구분되어 한눈에 보기 좋음
- 배경색 또는 구분선으로 시각화

### 2.3 재고 현황 섹션 (Inventory Status)
위치: 화면 중상단

#### 2.3.1 메뉴별 재고 표시
그리드 형태로 메뉴 재고 표시 (3개 열)

**각 메뉴 카드의 구성:**
- **메뉴명**: 
  - 아메리카노(ICE)
  - 아메리카노(HOT)
  - 카페라떼
  - 등 (확장 가능)
- **현재 재고 수량**: 
  - 예: "10개" (큰 텍스트로 강조)
- **수량 조절 버튼**:
  - 감소 버튼 (-)
  - 증가 버튼 (+)
  - 또는 input 필드에서 직접 입력 가능

#### 2.3.2 재고 관리 기능
- 각 메뉴의 현재 재고량 조회
- 재고 수량 실시간 수정 가능
- 버튼 클릭 또는 입력으로 수량 변경
- 변경사항 자동 저장 (또는 저장 버튼)

### 2.4 주문 현황 섹션 (Order Status)
위치: 화면 하단

#### 2.4.1 주문 목록 구성
테이블 또는 리스트 형식으로 주문정보 표시

**주문 항목당 표시되는 정보:**
- **주문 시간**: 
  - 예: "7월 31일 13:00"
- **주문 내용**: 
  - 메뉴명과 수량
  - 예: "아메리카노(ICE) x 1"
- **가격**: 
  - 주문의 총 가격
  - 예: "4,000원"
- **상태 변경 버튼**:
  - 주문 접수 (Receive Order)
  - 제조 시작 (Start Production)
  - 제조 완료 (Complete) 
  - 등의 상태 전환 버튼

#### 2.4.2 주문 상태 관리
- **주문 흐름**: 새 주문 → 접수 → 제조 중 → 완료
- **상태 변경**: 버튼 클릭으로 다음 단계로 진행
- **실시간 업데이트**: 상태 변경 시 대시보드 지표도 함께 변경

#### 2.4.3 주문 필터링 (추후 고려)
- 완료된 주문은 숨기거나 분리
- 진행 중인 주문 우선 표시

## 3. UI/UX 요구사항

### 3.1 레이아웃
- **상단**: 고정 네비게이션 헤더
- **상단-중앙**: 대시보드 요약 (4개 지표)
- **중앙**: 재고 현황 (3개 열 그리드)
- **하단**: 주문 현황 리스트 (스크롤 가능)

### 3.2 시각 디자인
- **색상**: 기본적인 흰색/회색 배경
- **재고 카드**: 경계선이 있는 박스 형식
- **버튼**: 명확한 액션 버튼 (예: +, -, 상태 변경 버튼)
- **폰트**: 기본 시스템 폰트
- **숫자 강조**: 재고 수량과 주문 지표를 크고 명확하게 표시

### 3.3 반응형 디자인
- 데스크톱 기준 (모바일 반응형은 추후 고려)

## 5. 사용자 플로우 (관리자)

### 5.1 기본 관리 플로우
1. 관리자가 '관리자' 탭으로 진입
2. **대시보드 조회**: 현재 주문 상태 한눈에 파악
3. **재고 확인**: 각 메뉴의 재고 수량 확인
4. **재고 조정**: 필요시 수량 수정 (예: 소진되면 0으로 변경)
5. **주문 관리**: 
   - 새로운 주문 확인
   - 순차적으로 상태 변경 (접수 → 제조 중 → 완료)
6. **대시보드 자동 갱신**: 모든 변경사항이 실시간 반영

### 5.2 주문 처리 단계
1. **신규 주문 접수**: "주문 접수" 버튼 클릭
2. **제조 시작**: "제조 중"으로 상태 변경 버튼 클릭
3. **제조 완료**: "제조 완료" 또는 "완료" 버튼 클릭

### 5.3 재고 관리 플로우
1. 재고 카드에서 현재 수량 확인
2. 필요시 +/- 버튼으로 수량 조정
3. 또는 직접 입력으로 수량 변경
4. 변경사항 저장

---

## 6. 페이지 상태

### 6.1 초기 상태 (Initial State)
- 대시보드 지표 로드됨
- 재고 목록 표시됨
- 주문 목록 표시됨 (있으면)

### 6.2 재고 수정 상태 (Editing Inventory)
- 재고 수량 입력/버튼 조작 중
- 변경 버튼 또는 자동 저장

### 6.3 주문 상태 변경 (Changing Order Status)
- 상태 변경 버튼 클릭
- 대시보드 지표 실시간 업데이트

### 6.4 로딩 상태 (Loading)
- 데이터 로딩 중 표시 (필요시)

### 6.5 에러 상태 (Error State)
- 재고 업데이트 실패
- 주문 상태 변경 실패

---

## 7. 에러 처리

### 7.1 예상 에러 시나리오
- 데이터 로드 실패 → 재시도 옵션 제공
- 재고 업데이트 실패 → 오류 메시지 표시 및 재시도
- 주문 상태 변경 실패 → 오류 메시지 표시

### 7.2 검증
- 재고 수량은 음수가 될 수 없음 (0 이상만 허용)
- 주문 상태는 정해진 순서대로만 변경 가능

---

## 8. 상태 전환 다이어그램

```
신규 주문 (Pending)
    ↓
주문 접수 (Received)
    ↓
제조 중 (In Progress)
    ↓
제조 완료 (Completed)
```

---

## 9. 성능 요구사항

### 9.1 실시간 성능
- 대시보드 지표 실시간 업데이트
- 재고 수량 변경 시 즉시 반영
- 주문 상태 변경 시 대시보드 자동 갱신

### 9.2 데이터 로딩
- 초기 화면 로드 시간: 2초 이내
- 재고 업데이트: 즉시 반영
