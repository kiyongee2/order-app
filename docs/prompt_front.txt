# 커피 주문 앱(프런트엔드)

## 1. 프로젝트 개요
## 1.1 프로젝트명
커피 주문 앱

### 1.2 프로젝트 목적
사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 
간단한 풀스택 웹 앱

### 1.3 개발 범위
- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능

## 2. 기술 스택
- 프런트엔드: HTML, CSS, 리엑트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL

## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음


# PRD(Product Requirements Document) - 제품 요구사항 문서

이제 프런트엔드 UI와 관련한 PRD를 작성할 거야

현재 앱에는 '주문하기'와 '관리자' 이렇게 2개의 화면이 있어,
우선 '주문하기' 화면의 와이어프레임을 첨부할 테니
'주문하기' 화면에 대한 PRD를 작성해줘

'관리자' 화면의 와이어프레임을 추가할 테니 '관리자' 화면에
대한 PRD도 작성해 줘

# 프런트엔드 - UI 폴더 생성
이제 프런트엔드 개발부터 시작할 거야
- 최상위 폴더에 ui 폴더를 만들고 그 곳에 프런트엔드 개발 환경을 만들어 줘.
- 리엑트와 바닐라 스크립트를 사용할 거야
- CRA 대신 Vite를 사용할 거야
- 리엑트 앱 개발 환경을 만들어 줘

## '주문하기' 화면 개발하기
개발 서버가 정상적으로 동작하는 것을 확인했어.
이제 PRD_주문하기_화면을 참고해서 커피 주문하기 화면을 만들어 줘.
첨부한 와이어프레임 이미지를 참고해서 화면을 만들어 줘
네비게이션과 버튼의 색상을 파랑색으로 맞춰 줘
커피 메뉴는 임의로 넣어 줘

### 수정할 부분
수정해야 할 부분은 다음과 같아
- 커피 이미지를 메뉴 이름에 맞도록 생성해 줘
- 전체 내용을 웹 브라우저 화면 중앙에 오도록 배치해 줘
- 네비게이션의 배경색은 버튼과 같은 색을 사용하고, 
  글자색은 배경에 맞게 수정해 줘
- 장바구니 영역을 2개로 나누어서 왼쪽에는 주문 내역을 표시하고
  오른쪽에는 총 금액과 '주문하기' 버튼을 넣어줘
- 같은 메뉴를 여러번 주문했을 때 장바구니에 반복해서 표시하지 말고
  개수를 늘려줘

## '관리자' 화면 만들기
이번에는 관리자 화면을 만들어 볼 거야
docs/PRD_관리자_화면.md 문서와 첨부한 그림을 참고해서 레이아웃을 만들어 줘
- '관리자 대시보드'에는 4개의 항목이 있고, 각 항목마다 몇 개가 있는지 표시할 거야.
- '재고 현황'에는 메뉴 3개에 대한 재고 개수를 표시할 거야.
  5개 미만이면 '주의', 0개면 '품절', 그 외에는 '정상'이라고 함께 표시해 줘
- '주문 현황'에는 접수된 주문 정보가 표시될 거야. 
  주문 접수 일자와 시간, 주문 메뉴, 금액 등을 함께 표시할 가야
- 주문이 들어오면 처음에는 '주문 접수' 상태이고, 관리자가 '제조 시작' 버튼을
  누르면 제조 과정으로 넘어가도록 할 거야
