# PRD - 주문하기 화면

## 1. 개요

### 1.1 화면명
주문하기 (Ordering Page)

### 1.2 목적
고객이 커피 메뉴를 선택하고, 옵션을 추가하여 주문할 수 있는 화면

### 1.3 우선순위
매우 높음 (주요 기능)

---

## 2. 기능 요구사항

### 2.1 상단 네비게이션 (Header)
- **앱 로고/이름**: "COZY" 표시
- **탭 메뉴**: 
  - "주문하기" (현재 화면)
  - "관리자" (관리자 화면으로 이동)
- **레이아웃**: 상단 고정 네비게이션

### 2.2 메뉴 섹션 (Menu Grid)
메뉴를 그리드 형태로 표시 (3개 열)

#### 2.2.1 메뉴 각 항목 구성요소
- **메뉴 이미지**: 커피 상품 이미지 표시
- **메뉴명**: 
  - 아메리카노(ICE) - 4,000원
  - 아메리카노(HOT) - 4,000원
  - 카페라떼 - 5,000원
- **설명**: 간단한 설명 텍스트 (예: "간단한 설명...")
- **옵션 선택**:
  - 샷 추가 (+500원) - 체크박스
  - 시럽 추가 (+0원) - 체크박스
- **담기 버튼**: 상품을 장바구니에 추가

### 2.3 옵션 기능 (Options)
#### 2.3.1 추가 옵션
- **샷 추가**: +500원 (선택 가능)
- **시럽 추가**: +0원 (선택 가능)

#### 2.3.2 옵션 선택 방식
- 각 메뉴별로 독립적인 옵션 선택
- 하나의 메뉴에 여러 옵션 중복 선택 가능
- 옵션 추가 시 가격 자동 계산

### 2.4 장바구니 섹션 (Cart)
위치: 화면 하단

#### 2.4.1 장바구니 구성요소
- **제목**: "장바구니"
- **주문 목록**: 
  - 상품명과 옵션 표시 (예: "아메리카노(ICE) (샷 추가) × 1")
  - 개별 가격 표시
- **총액 표시**: 모든 상품과 옵션의 합계 금액 (한눈에 보이는 위치)
- **주문하기 버튼**: 최종 주문 확정

#### 2.4.2 장바구니 동작
- 메뉴의 "담기" 버튼 클릭 시 아이템 추가
- 추가된 상품의 수량 조정 가능 (필요시)
- 장바구니가 비어있으면 주문하기 버튼 비활성화
- 주문하기 버튼 클릭 시 주문 실행

## 3. UI/UX 요구사항

### 3.1 레이아웃
- **상단**: 고정 네비게이션 헤더
- **중간**: 메뉴 그리드 (3개 열 레이아웃)
- **하단**: 장바구니 영역 (고정 또는 스크롤 가능)

### 3.2 시각 디자인
- **색상**: 기본적인 흰색/회색 배경
- **메뉴 카드**: 경계선이 있는 박스 형식
- **버튼**: 텍스트 기반 또는 아이콘 버튼
- **폰트**: 기본 시스템 폰트

### 3.3 반응형 디자인
- 데스크톱 기준 (모바일 반응형은 추후 고려)

## 5. 사용자 플로우

### 5.1 기본 주문 플로우
1. 사용자가 '주문하기' 탭으로 진입
2. 메뉴 목록 표시
3. 원하는 메뉴 선택 및 옵션 선택
4. "담기" 버튼 클릭 → 장바구니에 아이템 추가
5. 필요시 다른 메뉴도 추가
6. 장바구니에서 최종 확인
7. "주문하기" 버튼 클릭 → 주문 완료

### 5.2 장바구니 수정 (필요시)
- 장바구니에서 수량 조정
- 아이템 삭제 (필요시)

---

## 6. 페이지 상태

### 6.1 초기 상태 (Initial State)
- 메뉴 목록 로드됨
- 장바구니 비어있음
- "주문하기" 버튼 비활성화

### 6.2 메뉴 선택 상태 (Menu Selection)
- 사용자가 메뉴 및 옵션 선택 중

### 6.3 장바구니에 아이템 있음 (Items in Cart)
- "주문하기" 버튼 활성화
- 총액 실시간 업데이트

### 6.4 주문 완료 상태 (Order Completed)
- 주문 확인 메시지 표시
- 장바구니 초기화

---

## 7. 에러 처리

### 7.1 예상 에러 시나리오
- 메뉴 로드 실패 → 재시도 옵션 제공
- 주문 제출 실패 → 오류 메시지 표시 및 재시도

### 7.2 검증
- 장바구니가 비어있을 시 주문 버튼 클릭 불가
